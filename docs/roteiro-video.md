# Roteiro do Vídeo de Apresentação (5 minutos)

> Vídeo a ser publicado no **YouTube** conforme item **6** do Desafio de Entrega Intermediária.

---

## Configuração rápida da gravação

- **Duração-alvo:** 5 minutos (não passar disso — cortes ajudam)
- **Resolução:** 1080p (1920×1080)
- **Áudio:** fone com microfone ou microfone dedicado, sem barulho de fundo
- **Tela:** abrir o GitHub e o GitHub Pages em janelas grandes, fonte aumentada (Ctrl + "+")
- **Software sugerido:** OBS Studio (gratuito), Loom ou ScreenRec
- **Visibilidade no YouTube:** **Não listado** (recomendado para entrega acadêmica)

---

## Roteiro em blocos

### Bloco 1 · Abertura (0:00 – 0:30) — 30 segundos

**O que mostrar:** sua câmera (se quiser) ou a página inicial do GitHub Pages.

**O que falar:**

> "Oi! Eu sou o Lucca Mariz, aluno de Ciência de Dados e Machine Learning no CEUB.
> Neste vídeo eu apresento o **PortfolioHUB**, meu repositório no GitHub criado
> como parte da entrega intermediária da disciplina.
> Vou mostrar a estrutura, os projetos, a página publicada no GitHub Pages e
> como integrei tudo ao meu LinkedIn."

---

### Bloco 2 · Tour pelo Repositório (0:30 – 1:45) — 1m15s

**O que mostrar:** página do repositório no GitHub (`github.com/Ucazin/portfolio-hub`).

**O que falar:**

> "O repositório está organizado em pastas. Na raiz temos o `README.md`,
> que funciona como documento de boas-vindas, com badges, índice de projetos,
> contato e licença.
>
> Dentro temos três áreas principais:
> - **`projetos-academicos`** com os trabalhos da graduação
> - **`projetos-pessoais`** com projetos meus, fora do curso
> - **`docs`** com a apresentação e este roteiro
> - **`assets`** com os arquivos da página web publicada
>
> Cada projeto tem seu próprio `README.md`, com objetivo, conceitos aplicados,
> instruções de execução e tecnologias."

*Abra um dos READMEs de projeto e mostre rolando.*

---

### Bloco 3 · Projetos em Destaque (1:45 – 3:30) — 1m45s

**Apresente os 3 projetos rapidamente. Aproximadamente 35 segundos cada.**

#### Projeto 1 — Análise Exploratória de Vendas
> "O primeiro projeto é uma análise exploratória de dados em Python, com Pandas
> e Matplotlib. Ele gera um dataset fictício de vendas, calcula estatísticas
> descritivas, identifica o top de produtos e plota a evolução da receita ao longo
> do ano. É o ponto de partida de qualquer análise de dados séria."

*Mostre o código `analise.py` e um dos gráficos gerados.*

#### Projeto 2 — Classificação Iris com Machine Learning
> "O segundo é um pipeline supervisionado completo com scikit-learn. Comparo
> três modelos — KNN, Regressão Logística e Random Forest — no dataset Iris,
> com normalização, split estratificado e avaliação por matriz de confusão.
> É o esqueleto que dá pra reaproveitar para qualquer problema de classificação."

*Mostre o terminal com a saída e a matriz de confusão.*

#### Projeto 3 — Dashboard COVID Interativo
> "O terceiro é um dashboard web em HTML, CSS e JavaScript puro, com Chart.js.
> Mostra 4 KPIs e três tipos de gráficos. É 100% estático, então roda direto
> no GitHub Pages, sem servidor."

*Abra a página do dashboard.*

---

### Bloco 4 · GitHub Pages (3:30 – 4:10) — 40 segundos

**O que mostrar:** a página principal publicada (`https://ucazin.github.io/portfolio-hub/`).

**O que falar:**

> "A página principal do portfólio está publicada no GitHub Pages. O tema é
> dark tech, com layout responsivo. Tem hero com chamada, seção de projetos,
> sobre, stack e contato.
>
> Tudo foi feito em HTML, CSS e JavaScript puros — sem frameworks. Isso garante
> que o site carregue rápido e funcione bem em qualquer dispositivo."

*Mostre o site em desktop e depois redimensione o navegador para simular mobile.*

---

### Bloco 5 · Versionamento e LinkedIn (4:10 – 4:45) — 35 segundos

**O que mostrar:** aba "Tags" do GitHub e seu perfil no LinkedIn.

**O que falar:**

> "Apliquei versionamento semântico — esta é a release `v1.0.0`, marcada com tag.
> Os commits seguem o padrão **Conventional Commits**, então o histórico fica
> fácil de ler.
>
> No LinkedIn, adicionei o repositório na seção 'Projetos' do meu perfil, com
> link direto pra esta página. Assim recrutadores conseguem ver tudo em poucos
> cliques."

*Mostre a seção "Projetos" do LinkedIn.*

---

### Bloco 6 · Encerramento (4:45 – 5:00) — 15 segundos

**O que falar:**

> "É isso. O PortfolioHUB vai continuar crescendo a cada nova matéria.
> O código está aberto, com licença MIT, em **github.com/Ucazin/portfolio-hub**.
> Obrigado por assistir!"

## Checklist antes de gravar

- [ ] Repositório está **público** no GitHub
- [ ] **GitHub Pages** está acessível e abrindo bem
- [ ] **LinkedIn** está com o projeto adicionado
- [ ] Janela do navegador limpa (sem extensões aparecendo, sem outras abas confidenciais)
- [ ] Áudio testado (gravar 10s primeiro e ouvir)
- [ ] Notificações do sistema **desativadas** durante a gravação
- [ ] Tem água por perto (gravação contínua resseca a garganta)

## Checklist depois de gravar

- [ ] Vídeo cortado dentro de 5 minutos
- [ ] Áudio audível e sem ruído alto
- [ ] Subido no YouTube como **"Não listado"**
- [ ] Título sugerido: `PortfolioHUB — Apresentação · Lucca Mariz · CEUB 2026`
- [ ] Descrição com links do repo, GitHub Pages e LinkedIn
- [ ] Link copiado para o PDF de entrega final
